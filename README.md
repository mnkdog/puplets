# Collaboration Readme for Claude

## Overview
This document defines the workflow and expectations for collaborating with Claude as a coding partner to build a simple static website. The purpose is to ensure clarity, trust, and quality delivery.

## Ways of Working

### 1. BDD Scenarios
- Use Gherkin syntax for Behavior-Driven Development (BDD).
- Claude will assist by asking clarifying questions to help generate precise BDD scenarios describing the website’s behavior.

### 2. Technology Advice
- Claude will recommend simple, free hosting-friendly technologies (e.g., Firebase) suitable for a static business card website redirecting visitors to an online store.
- Recommendations should consider ease of use, testing integration, and scalability.

### 3. TDD Testing
- Claude will convert BDD scenarios into full coverage Test-Driven Development (TDD) tests.
- Testing includes unit tests, integration tests (if applicable), and UI tests to minimize future risk when making changes.

### 4. Development and Version Control
- Follow trunk-based development with very small, micro commits.
- Feature branches to be short-lived (maximum one day).
- Push to main branch only when all tests pass.
- Use your preferred version control system (e.g., Git).

### 5. Preview and Feedback
- Claude will provide website previews using local builds or demo URLs.
- Review and feedback will occur after implementing each BDD scenario.
- Claude may advise on adjusting review frequency if beneficial.

### 6. Logo and Design
- Claude will use the provided PNG logo file.
- Claude will propose a basic design and advise on logo placement in the website for branding and usability.

### 7. Communication
- Claude will provide regular status updates and progress summaries.
- Before continuing with new work, Claude will agree on the amount of work to do without updates.
- You will respond promptly to Claude’s questions to avoid bottlenecks.

### 8. Hosting and Deployment
- Claude will automate deployment to free hosting service (e.g., Firebase) on successful main branch pushes.

## Summary
This collaboration promotes incremental delivery, full test coverage, and continuous feedback to build a stable, maintainable website with transparent communication and clear expectations.
